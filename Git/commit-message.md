# Git Commit Message

## 규칙
- 제목과 본문을 한 줄 띄워 분리하기
- 제목은 영문 기준 50자 이내로
- 제목 첫 글자를 대문자로
- 제목 끝에 '.' 금지
- 제목은 명령조로
- 본문은 영문 기준 72자마다 줄 바꾸기
- 본문은 '무엇을', '왜'에 맞춰 작성하기

## 타입
- feat: 기능 구현
- refactor: 코드 리펙토링
- test: 테스트 추가, 테스트 리팩토링
- docs: 문서 수정
- style: 코드 포맷, 세미콜론 누락 등 코드 수정이 없는 경우
- fix: 버그 고침
- rename: 파일, 패키지명 수정
- remove: 파일 삭제
- comment: 주석 관련
- chore: 그 외 자잘한 수정
